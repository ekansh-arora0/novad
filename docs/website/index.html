NOVAD

FTC Defense Library

Stop getting pushed around. Two methods. That's it.

Get Started View on GitHub

Introduction

Novad is a defense library for FTC robots. It uses odometry to detect when your robot is being pushed and automatically applies counter-force to resist movement.

How it Works

Every loop iteration, Novad:

  1. Reads your joystick input - What direction did you command?
  2. Reads odometry - What direction did the robot actually move?
  3. Compares them - Any difference is an external force (being pushed)
  4. Applies PIDF correction - Counter-force to resist the push

The result: You drive normally, but your robot resists being pushed around.

Two Modes

üõ°Ô∏è Defense Mode

Drive while resisting pushes. Use novad.defense() instead of your normal drive call.

üîí Lockdown Mode

Become completely immovable. Use novad.lockdown() when you want to plant.

Requirements

Prerequisites

  • Mecanum drivetrain - 4 mecanum wheels
  • Odometry - GoBilda Pinpoint (recommended) or Three Dead Wheels
  • FTC SDK 9.0+

Step 2: Configure Constants.java

Edit Constants.java to match your robot. It looks just like Pedro Pathing's constants:

public class Constants {

    // PIDF values (copy from Pedro if you have them!)
    public static PIDFCoefficients translationalPIDF = 
        new PIDFCoefficients(0.065, 0, 0.033, 0.001);
    public static PIDFCoefficients headingPIDF = 
        new PIDFCoefficients(1, 0, 0, 0.01);

    // Drivetrain (same format as Pedro!)
    public static MecanumConstants driveConstants = new MecanumConstants()
        .leftFrontMotorName("front_left_drive")
        .leftRearMotorName("back_left_drive")
        .rightFrontMotorName("front_right_drive")
        .rightRearMotorName("back_right_drive")
        .leftFrontMotorDirection(Direction.FORWARD)
        .leftRearMotorDirection(Direction.FORWARD)
        .rightFrontMotorDirection(Direction.FORWARD)
        .rightRearMotorDirection(Direction.FORWARD);

    // Pinpoint (same format as Pedro!)
    public static PinpointConstants localizerConstants = new PinpointConstants()
        .hardwareMapName("pinpoint")
        .encoderResolution(GoBildaOdometryPods.goBILDA_4_BAR_POD)
        .forwardEncoderDirection(EncoderDirection.FORWARD)
        .strafeEncoderDirection(EncoderDirection.FORWARD);

    // Create Novad
    public static Novad createNovad(HardwareMap hardwareMap) {
        return new NovadBuilder(hardwareMap)
            .mecanumDrivetrain(driveConstants)
            .pinpointLocalizer(localizerConstants)
            .translationalPIDF(translationalPIDF)
            .headingPIDF(headingPIDF)
            .build();
    }
}

Step 3: Use in Your TeleOp

package org.firstinspires.ftc.teamcode;

import com.qualcomm.robotcore.eventloop.opmode.LinearOpMode;
import com.qualcomm.robotcore.eventloop.opmode.TeleOp;
import org.firstinspires.ftc.teamcode.novad.Constants;
import org.firstinspires.ftc.teamcode.novad.Novad;

@TeleOp(name = "TeleOp with Defense")
public class MyTeleOp extends LinearOpMode {

    @Override
    public void runOpMode() {
        // Initialize Novad (just like Pedro's createFollower!)
        Novad novad = Constants.createNovad(hardwareMap);

        waitForStart();

        while (opModeIsActive()) {
            // Drive with defense
            novad.defense(
                gamepad1.left_stick_x,     // strafe
                -gamepad1.left_stick_y,    // forward (negate Y!)
                gamepad1.right_stick_x     // rotate
            );

            // Optional: lockdown on button
            if (gamepad1.a) novad.lockdown();
            if (gamepad1.b) novad.unlock();

            telemetry.addData("Locked", novad.isLocked());
            telemetry.update();
        }
    }
}

‚ö†Ô∏è Important: Negate the Y Axis!

Gamepad Y is inverted - pushing UP gives a negative value. You must negate it: -gamepad1.left_stick_y

Tuning

Tuning Novad has three steps:

1Setup

Configure motor names and directions.

2Localization

Set up your odometry system.

3PID Tuning

Tune the defense strength.

Step 1: Setup

In NovadSetup.java, configure your motor names and directions.

Motor Names

These must exactly match the names in your robot configuration (Driver Station ‚Üí Configure Robot).

public static String FRONT_LEFT  = "frontLeft";
public static String FRONT_RIGHT = "frontRight";
public static String BACK_LEFT   = "backLeft";
public static String BACK_RIGHT  = "backRight";

Motor Directions

Standard mecanum: left side = REVERSE, right side = FORWARD. If a wheel spins the wrong way, flip that motor's direction.

public static Direction FRONT_LEFT_DIR  = Direction.REVERSE;
public static Direction FRONT_RIGHT_DIR = Direction.FORWARD;
public static Direction BACK_LEFT_DIR   = Direction.REVERSE;
public static Direction BACK_RIGHT_DIR  = Direction.FORWARD;

üí° Tip: Testing Motor Directions

Push the left stick forward. All wheels should spin to move the robot forward. If the robot moves backward, flip all four directions. If only one wheel is wrong, flip just that one.

Step 2: Localization

Novad needs to know where your robot is on the field. Choose your localizer below.

GoBilda Pinpoint

Recommended. Self-contained odometry computer. Easiest setup.

Three Dead Wheels

Traditional setup with three odometry wheels and encoders.

Pinpoint Localizer

Prerequisites

Ensure two odometry pods are connected to the Pinpoint: one parallel to your chassis (forward/X) and one perpendicular (lateral/Y). The Pinpoint should be connected to a REV Hub via I2C (not port 0, which has the IMU).

In NovadSetup.java, set:

public static boolean USE_PINPOINT = true;
public static boolean USE_THREE_WHEEL = false;

// Device name in your robot configuration
public static String PINPOINT_NAME = "pinpoint";

Encoder Resolution

If you're using the standard goBilda odometry pods, leave this as-is:

public static boolean PINPOINT_USE_GOBILDA_PODS = true;

If you're using custom encoders (not goBilda pods), set PINPOINT_USE_GOBILDA_PODS = false and configure:

public static double PINPOINT_TICKS_PER_MM = 19.89;  // Your encoder's ticks per mm

Pod Offsets

You must tell the Pinpoint where your odometry pods are located relative to your robot's center of rotation. Measure in millimeters.

// Pod offsets from robot center (MILLIMETERS)
// X_OFFSET: Forward = positive, Backward = negative
// Y_OFFSET: Left = positive, Right = negative
public static double PINPOINT_X_OFFSET = -50;  // Example: X pod is 50mm behind center
public static double PINPOINT_Y_OFFSET = 120;  // Example: Y pod is 120mm left of center

Encoder Directions

Test your encoder directions by running your TeleOp and pushing the robot:

  1. Push the robot forward - X should increase
  2. Push the robot left - Y should increase

If either direction is wrong, reverse that encoder:

public static boolean PINPOINT_X_REVERSED = false;
public static boolean PINPOINT_Y_REVERSED = false;

üìå Pinpoint Does NOT Need ticks_to_inches, track_width, or center_offset!

Unlike three-wheel dead wheel setups, the Pinpoint handles all encoder math internally. You only need to configure:

  • Pod offsets (X_OFFSET, Y_OFFSET in mm)
  • Encoder resolution (goBilda pods or custom)
  • Encoder directions (if counting backwards)

The Pinpoint computes position, heading, and velocity automatically.

Three Wheel Localizer

Prerequisites

Ensure three odometry pods are connected: two parallel to your chassis length (left and right) and one perpendicular (center/strafe). Encoders should be plugged into motor encoder ports. Ports 0 and 3 on REV Hubs are fastest.

In NovadSetup.java, set:

public static boolean USE_PINPOINT = false;
public static boolean USE_THREE_WHEEL = true;

Encoder Ports

Dead wheel encoders plug into motor encoder ports. Enter the name of the motor whose port each encoder uses:

public static String LEFT_ENCODER   = "frontLeft";   // Left parallel pod
public static String RIGHT_ENCODER  = "frontRight";  // Right parallel pod
public static String CENTER_ENCODER = "backLeft";    // Perpendicular pod

Ticks to Inches

This critical value converts encoder ticks to real-world inches.

// Formula: (œÄ √ó wheel_diameter_inches) / encoder_CPR

üí° Tip: Calibrating Ticks to Inches

Push your robot exactly 48 inches (2 field tiles). Read the encoder count. Then: TICKS_TO_INCHES = 48.0 / encoder_count. Run multiple times and average for best accuracy.

Physical Measurements

Measure these distances on your robot in inches:

// Distance between left and right encoder wheels (center to center)
public static double TRACK_WIDTH = 14.0;

// Distance from robot center to center encoder
// Positive = forward of center, Negative = behind center
public static double CENTER_OFFSET = 6.0;

Encoder Directions

Test by pushing the robot forward - both parallel encoders should count in the same direction. Push left - center encoder should count positive.

public static boolean LEFT_ENCODER_REVERSED   = false;
public static boolean RIGHT_ENCODER_REVERSED  = false;
public static boolean CENTER_ENCODER_REVERSED = false;

Step 3: PID Tuning

Novad uses PIDF controllers to resist being pushed. There are two controllers:

  • Translational PIDF - Controls X and Y position (resists being pushed sideways/forward/backward)
  • Heading PIDF - Controls rotation (resists being spun)

PIDF Explained

TermWhat it DoesIf Too HighIf Too Low
P (Proportional)Main correction strength. Pushes back proportionally to how far you've been pushed.Robot oscillates/shakesRobot doesn't resist enough
I (Integral)Fixes slow drift over time. Accumulates error and pushes harder.Robot overshoots, oscillatesRobot slowly drifts
D (Derivative)Dampens oscillation. Resists rapid changes.Robot feels sluggishRobot shakes/oscillates
F (Feedforward)Constant power to overcome static friction.Robot jitters when stillMotors buzz at low corrections

Default Values

// Translational (X/Y position)
public static double TRANS_P = 0.1;
public static double TRANS_I = 0.0;
public static double TRANS_D = 0.01;
public static double TRANS_F = 0.0;

// Heading (rotation)
public static double HEADING_P = 1.0;
public static double HEADING_I = 0.0;
public static double HEADING_D = 0.01;
public static double HEADING_F = 0.0;

Live Tuning with FTC Dashboard

All values in NovadSetup.java are marked with @Config, so you can tune them in real-time:

  1. Connect your laptop to the robot's WiFi
  2. Open http://192.168.43.1:8080/dash
  3. Find "NovadSetup" in the sidebar
  4. Adjust values - changes apply instantly!
  5. Once happy, copy final values back to NovadSetup.java

Tuning Procedure

Step-by-Step

  1. Start with defaults - They work for most robots
  2. Test defense mode - Drive around while someone pushes you
  3. If robot oscillates (shakes) - Decrease P, increase D
  4. If robot doesn't resist enough - Increase P
  5. If motors buzz at low corrections - Add small F value (0.02-0.05)
  6. If robot slowly drifts - Add small I value (0.001)
  7. Test lockdown - Should be rock solid. If it shakes, increase D

Other Settings

// Maximum correction power (0.0 to 1.0)
public static double MAX_POWER = 0.8;

// Joystick deadzone
public static double DEADZONE = 0.05;

// Field-centric driving (requires good heading from odometry)
public static boolean USE_FIELD_CENTRIC = false;

// Brake mode when stopped (helps hold position)
public static boolean USE_BRAKE_MODE = true;

API Reference

Defense Mode

novad.defense(double strafe, double forward, double rotate)

The main method - call this every loop iteration.

This drives your robot while actively resisting external pushes. It compares driver input vs actual movement (via odometry). Any discrepancy is treated as an external push, and Novad applies counter-force.

Parameters:

  • strafe - Left stick X. Range -1 to 1. Positive = right.
  • forward - Left stick Y. Range -1 to 1. Positive = forward. Negate gamepad Y!
  • rotate - Right stick X. Range -1 to 1. Positive = turn right.

What it resists:

  • ‚úì Being pushed sideways
  • ‚úì Being pushed forward/backward
  • ‚úì Being spun/rotated

Lockdown Mode

novad.lockdown()

Makes the robot completely immovable.

Call this when you want to plant and block. The robot holds its exact position (X, Y, AND heading) with maximum force. Driver input is ignored while locked.

Use case: You're in a good defensive position and want to become an immovable wall.

novad.unlock()

Exits lockdown mode.

Returns to normal defense mode where you can drive while resisting pushes.

boolean novad.isLocked()
Returns true if currently in lockdown mode. Useful for telemetry.

Helper Methods

Pose2d novad.getPose()
Returns current robot position as Pose2d(x, y, heading). X/Y in inches, heading in radians.
void novad.resetHeading()
Resets the heading to zero. Useful at the start of TeleOp for field-centric driving.

Factory Method

static Novad Novad.create(HardwareMap hardwareMap)
Creates a Novad instance using settings from NovadSetup.java. Call once in your init.

Troubleshooting

Robot doesn't resist pushes at all

  • Check that odometry is working (use novad.getPose() in telemetry)
  • Ensure USE_PINPOINT or USE_THREE_WHEEL is set to true
  • Increase TRANS_P value

Robot oscillates/shakes

  • Decrease TRANS_P
  • Increase TRANS_D
  • Same for heading: decrease HEADING_P, increase HEADING_D

Motors buzz/jitter at low corrections

  • Add small TRANS_F value (try 0.03)
  • This overcomes static friction

Robot slowly drifts

  • Add small TRANS_I value (try 0.001)
  • Check odometry accuracy

Lockdown doesn't hold

  • Increase TRANS_P and HEADING_P
  • Ensure USE_BRAKE_MODE = true
  • Check that odometry is accurate

Position drifts in one direction

  • Check encoder directions (X forward = increase, Y left = increase)
  • For Pinpoint: verify PINPOINT_X_REVERSED / PINPOINT_Y_REVERSED
  • For Three-Wheel: verify encoder reversed settings and offsets

Pinpoint not detected

  • Ensure it's connected to I2C port (not port 0, the IMU is there)
  • Check the device name matches PINPOINT_NAME exactly
  • The Pinpoint must be configured in your robot configuration
