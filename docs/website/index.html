NOVAD

FTC Defense Library v2.2.0

Stop getting pushed around. Three methods. Two modes. That's it.

Get Started View on GitHub

Introduction

Novad is a defense library for FTC robots. It uses odometry to detect when your robot is being pushed and automatically applies counter-force to resist movement.

How it Works

Every loop iteration (~5ms), Novad:

  1. Reads your motor commands - What did you tell the motors to do?
  2. Reads odometry - What did the robot actually do?
  3. Compares them - Any difference is an external force (being pushed)
  4. Applies PIDF correction - Counter-force to resist the push

The result: You drive normally, but your robot resists being pushed around.

Two Modes

üõ°Ô∏è Defense Mode

Drive while resisting pushes. Use novad.defense() or novad.defenseWithMotors().

üîí Lockdown Mode

Become completely immovable. Driver input is IGNORED. Use novad.lockdown().

Step 2: Configure Constants.java

Edit Constants.java to match your robot. It looks just like Pedro Pathing's constants:

public class Constants {

    // PIDF values (copy from Pedro if you have them!)
    public static PIDFCoefficients translationalPIDF = 
        new PIDFCoefficients(0.065, 0, 0.033, 0.001);
    public static PIDFCoefficients headingPIDF = 
        new PIDFCoefficients(1, 0, 0, 0.01);

    // Drivetrain (same format as Pedro!)
    public static MecanumConstants driveConstants = new MecanumConstants()
        .leftFrontMotorName("front_left_drive")
        .leftRearMotorName("back_left_drive")
        .rightFrontMotorName("front_right_drive")
        .rightRearMotorName("back_right_drive")
        .leftFrontMotorDirection(Direction.FORWARD)
        .leftRearMotorDirection(Direction.FORWARD)
        .rightFrontMotorDirection(Direction.FORWARD)
        .rightRearMotorDirection(Direction.FORWARD);

    // Pinpoint (same format as Pedro!)
    public static PinpointConstants localizerConstants = new PinpointConstants()
        .hardwareMapName("pinpoint")
        .encoderResolution(GoBildaOdometryPods.goBILDA_4_BAR_POD)
        .forwardEncoderDirection(EncoderDirection.FORWARD)
        .strafeEncoderDirection(EncoderDirection.FORWARD);

    // Create Novad
    public static Novad createNovad(HardwareMap hardwareMap) {
        return new NovadBuilder(hardwareMap)
            .mecanumDrivetrain(driveConstants)
            .pinpointLocalizer(localizerConstants)
            .translationalPIDF(translationalPIDF)
            .headingPIDF(headingPIDF)
            .build();
    }
}

Step 3: Use in Your TeleOp

package org.firstinspires.ftc.teamcode;

import com.qualcomm.robotcore.eventloop.opmode.LinearOpMode;
import com.qualcomm.robotcore.eventloop.opmode.TeleOp;
import org.firstinspires.ftc.teamcode.novad.Constants;
import org.firstinspires.ftc.teamcode.novad.Novad;

@TeleOp(name = "TeleOp with Defense")
public class MyTeleOp extends LinearOpMode {

    @Override
    public void runOpMode() {
        // Initialize Novad (just like Pedro's createFollower!)
        Novad novad = Constants.createNovad(hardwareMap);

        waitForStart();

        while (opModeIsActive()) {
            // Drive with defense
            novad.defense(
                gamepad1.left_stick_x,     // strafe
                -gamepad1.left_stick_y,    // forward (negate Y!)
                gamepad1.right_stick_x     // rotate
            );

            // Optional: lockdown on button
            if (gamepad1.a) novad.lockdown();
            if (gamepad1.b) novad.unlock();

            telemetry.addData("Locked", novad.isLocked());
            telemetry.update();
        }
    }
}

‚ö†Ô∏è Important: Negate the Y Axis!

Gamepad Y is inverted - pushing UP gives a negative value. You must negate it: -gamepad1.left_stick_y

Tuning

Tuning Novad has three steps:

1Setup

Configure motor names and directions in Constants.java.

2Localization

Configure your Pinpoint odometry.

3PID Tuners

Tune translational and heading PIDF.

Step 1: Setup

In Constants.java, configure your drivetrain. This looks exactly like Pedro Pathing's setup.

Drivetrain Constants

Set your motor names and directions. Names must exactly match your robot configuration.

public static MecanumConstants driveConstants = new MecanumConstants()
    .leftFrontMotorName("front_left_drive")
    .leftRearMotorName("back_left_drive")
    .rightFrontMotorName("front_right_drive")
    .rightRearMotorName("back_right_drive")
    .leftFrontMotorDirection(DcMotorSimple.Direction.FORWARD)
    .leftRearMotorDirection(DcMotorSimple.Direction.FORWARD)
    .rightFrontMotorDirection(DcMotorSimple.Direction.FORWARD)
    .rightRearMotorDirection(DcMotorSimple.Direction.FORWARD)
    .useBrakeMode(true);

üí° Testing Motor Directions

Push the left stick forward. All wheels should spin to move the robot forward. If the robot moves backward, flip all four directions. If only one wheel is wrong, flip just that one.

Step 2: Localization

Prerequisites

Ensure two odometry pods are connected to the Pinpoint: one parallel to your chassis (forward/X) and one perpendicular (strafe/Y). The Pinpoint should be connected to a REV Hub via I2C (not port 0).

Pinpoint Constants

Configure your Pinpoint localizer. This matches Pedro Pathing's format exactly:

public static PinpointConstants localizerConstants = new PinpointConstants()
    .hardwareMapName("pinpoint")
    .encoderResolution(GoBildaPinpointDriver.GoBildaOdometryPods.goBILDA_4_BAR_POD)
    .forwardEncoderDirection(GoBildaPinpointDriver.EncoderDirection.FORWARD)
    .strafeEncoderDirection(GoBildaPinpointDriver.EncoderDirection.FORWARD);

Encoder Resolution

If using goBilda odometry pods, use goBILDA_4_BAR_POD. For custom encoders:

.customEncoderResolution(19.89)  // ticks per mm

Encoder Directions

Test by pushing the robot:

  1. Push forward ‚Üí X should increase
  2. Push left ‚Üí Y should increase

If either is wrong, change that encoder to EncoderDirection.REVERSED.

üìå No Pod Offsets Needed!

Unlike Pedro Pathing, Novad does NOT require pod offsets. The Pinpoint handles position internally. Novad only cares about changes in position (deltas), not absolute positioning.

Step 3: PID Tuners

Novad uses two PIDF controllers:

  • Translational PIDF - Resists being pushed in X/Y
  • Heading PIDF - Resists being rotated

If You Already Use Pedro Pathing

üí° Copy Your Pedro Values!

If you've already tuned Pedro Pathing, copy your translational and heading PIDF values directly:

// Into Novad Constants:
public static PIDFCoefficients translationalPIDF = 
    new PIDFCoefficients(0.065, 0, 0.033, 0.001);
public static PIDFCoefficients headingPIDF = 
    new PIDFCoefficients(1, 0, 0, 0.01);

Starting From Scratch

Use these recommended starting values:

public static PIDFCoefficients translationalPIDF = 
    new PIDFCoefficients(0.1, 0, 0.02, 0);
public static PIDFCoefficients headingPIDF =

Tuning Procedure

Step-by-Step

  1. Start with defaults or copy from Pedro
  2. Test defense mode - Drive while someone pushes you
  3. If robot oscillates - Decrease P, increase D
  4. If robot doesn't resist - Increase P
  5. If motors buzz - Add small F (0.001-0.01)
  6. Test lockdown - Should be rock solid

API Reference

Novad v2.2.0 provides three ways to drive with defense, plus lockdown mode.

TL;DR - Which Method Do I Use?

üõ°Ô∏è Defense Mode

Defense mode lets you drive normally while resisting pushes. Use ONE of these methods every loop.

novad.defense(Gamepad gamepad)

RECOMMENDED - Automatically reads all inputs.

Pass your gamepad directly. Novad reads joysticks and handles field-centric transformation automatically.

// Simplest usage
novad.defense(gamepad1);

Limitation: Won't catch button-triggered macros (D-pad moves, auto-align, etc.)

novad.defense(double strafe, double forward, double rotate)

Pass joystick values directly.

Use this if you need to modify joystick values before passing them.

// Manual joystick values
novad.defense(
    gamepad1.left_stick_x,     // strafe (-1 = left, +1 = right)
    -gamepad1.left_stick_y,    // forward (NEGATE! -1 = back, +1 = forward)
    gamepad1.right_stick_x     // rotate (-1 = CCW, +1 = CW)
);

Limitation: Won't catch button-triggered movements!

novad.defenseWithMotors(double lf, double rf, double lb, double rb)

BEST FOR CUSTOM CONTROLS - Pass intended motor powers.

Use this if your code has button-triggered movements (D-pad macros, auto-align, auto-score positions). Novad tracks what the motors are commanded to do vs what actually happens.

// Calculate motor powers yourself
double strafe = gamepad1.left_stick_x;
double forward = -gamepad1.left_stick_y;
double rotate = gamepad1.right_stick_x;

// Button-triggered movements (the edge case!)
if (gamepad1.dpad_up) { forward = 0.5; strafe = 0; rotate = 0; }
if (gamepad1.dpad_down) { forward = -0.5; strafe = 0; rotate = 0; }
if (gamepad1.dpad_left) { strafe = -0.5; forward = 0; rotate = 0; }
if (gamepad1.dpad_right) { strafe = 0.5; forward = 0; rotate = 0; }

// Calculate mecanum motor powers
double lf = forward + strafe + rotate;
double rf = forward - strafe - rotate;
double lb = forward - strafe + rotate;
double rb = forward + strafe - rotate;

// Normalize if any > 1
double max = Math.max(Math.abs(lf), Math.max(Math.abs(rf), 
             Math.max(Math.abs(lb), Math.abs(rb))));
if (max > 1.0) { lf /= max; rf /= max; lb /= max; rb /= max; }

// Pass to Novad - it will add corrections!
novad.defenseWithMotors(lf, rf, lb, rb);

How it works: Novad tracks what movement those motor powers should cause, compares to what actually happened (via odometry), and applies corrections for any discrepancy.

üîí Lockdown Mode

Lockdown mode makes your robot completely immovable. It holds position with maximum force and ignores driver input.

novad.lockdown()

Makes the robot immovable at its current position.

When called, Novad saves the current X, Y, and heading. It then actively fights to stay at that exact position. Driver input is completely ignored while locked.

// Bind lockdown to a button
if (gamepad1.a) {
    novad.lockdown();  // Press and HOLD to stay locked
}

When to use:

  • You're in a good blocking position and want to plant
  • Opponent is trying to push you out of the way
  • You need to hold position while scoring

Note: Keep calling lockdown() every loop to stay locked. If you stop calling it, call unlock() to exit cleanly.

novad.unlock()

Exits lockdown mode and returns to defense mode.

if (gamepad1.b) {
    novad.unlock();  // Release to resume driving
}

After unlocking, you can resume driving with defense().

boolean novad.isLocked()

Returns true if currently in lockdown mode. Useful for telemetry or conditional logic.

telemetry.addData("Mode", novad.isLocked() ? "üîí LOCKED" : "üõ°Ô∏è DEFENSE");

üîß Helper Methods

Pose2d novad.getPose()

Returns current robot position as Pose2d(x, y, heading).

  • x - Position in inches (forward/back from start)
  • y - Position in inches (left/right from start)
  • heading - Angle in radians (0 = starting direction)
Pose2d pose = novad.getPose();
telemetry.addData("Position", "X: %.1f, Y: %.1f, H: %.1f¬∞", 
    pose.x, pose.y, Math.toDegrees(pose.heading));
void novad.resetHeading()

Sets current heading as "zero". Call at the start of TeleOp for field-centric driving.

// At start of TeleOp
waitForStart();
novad.resetHeading();  // Current direction is now "forward"
void novad.resetPosition()

Resets position to (0, 0, 0). Useful if odometry drifted.

double novad.getLoopTimeMs()

Returns time since last update in milliseconds. Use for debugging - should be under 20ms.

telemetry.addData("Loop", "%.1f ms", novad.getLoopTimeMs());
void novad.setTranslationalPIDF(PIDFCoefficients pidf)

Update translational PIDF at runtime. Useful for live tuning.

void novad.setHeadingPIDF(PIDFCoefficients pidf)

Update heading PIDF at runtime. Useful for live tuning.

Creates a Novad instance using settings from Constants.java. Call once during init.

Troubleshooting

Robot doesn't resist pushes at all

  • Check that odometry is working (use novad.getPose() in telemetry)
  • Ensure USE_PINPOINT or USE_THREE_WHEEL is set to true
  • Increase TRANS_P value

Robot oscillates/shakes

  • Decrease TRANS_P
  • Increase TRANS_D
  • Same for heading: decrease HEADING_P, increase HEADING_D

Motors buzz/jitter at low corrections

  • Add small TRANS_F value (try 0.03)
  • This overcomes static friction

Robot slowly drifts

  • Add small TRANS_I value (try 0.001)
  • Check odometry accuracy

Lockdown doesn't hold

  • Increase TRANS_P and HEADING_P
  • Ensure USE_BRAKE_MODE = true
  • Check that odometry is accurate

Position drifts in one direction

  • Check encoder directions (X forward = increase, Y left = increase)
  • For Pinpoint: verify PINPOINT_X_REVERSED / PINPOINT_Y_REVERSED
  • For Three-Wheel: verify encoder reversed settings and offsets

Pinpoint not detected

  • Ensure it's connected to I2C port (not port 0, the IMU is there)
  • Check the device name matches PINPOINT_NAME exactly
  • The Pinpoint must be configured in your robot configuration
